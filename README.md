# 装修需求记录

这是一个用于整理室内装修需求的 Obsidian Vault。当前内容不是软件项目，而是围绕新房装修进行的需求记录、生活场景梳理、家具家电资料收集和后续设计分析准备。

## 当前阶段

目前处在日常需求记录阶段，主要通过 diary study 的方式记录真实生活场景，包括：

- 每天在家中的行为动线
- 各房间中的使用场景
- 家具、家电、生活用品的需求和备选项
- 收纳、照明、插座、清洁、宠物照看等痛点
- 对喜欢的家具和产品的图片、链接、备注

在明确开始之前，暂时不进入完整的需求归纳、journey map、persona 或机会点分析阶段。

## 主要入口

- `场景需求分析.md`：当前最核心的生活场景记录，包含租房现状、新房户型、工作日早晨、上班期间、下班回家等日常动线。
- `喜欢的家具记录.md`：喜欢的家具、灯具和产品记录，包含外部链接、简短理由和本地图片。
- `房间/`：按空间整理的痛点、场景、照明、存储、插座需求和问题。
- `家具/`：床、床头柜、床头灯、马桶、Nychair X 等家具条目。
- `家电/`：呼吸机、扫地机器人、灯具、空调、冰箱、电视、智能门锁、宠物相关设备等家电条目。
- `生活用品/`：耳塞、洗手液、湿纸巾、矿泉水等生活用品条目。
- `家紡/`：枕头等家纺条目。
- `宠物.md`、`智能家居.md`：宠物照看和智能家居相关记录入口。
- `关系图.canvas`：Obsidian Canvas，用于查看资料之间的关系。
- `Agent.md`：AI 协作上下文，说明当前阶段、协作方式、图片读取规则和信息准确性原则。

## 图片规则

图片统一存放在 `src/` 文件夹下，并通过 Obsidian 图片语法引用：

```markdown
![[图片文件名|alt文本|显示宽度]]
```

当前约定：

- `图片文件名` 对应 `src/图片文件名`
- `alt文本` 与图片文件名一致
- `显示宽度` 只影响 Obsidian 展示，不属于文件路径
- 图片是需求资料的一部分，review 时需要结合图片本身、alt 文本和图片附近的文字说明一起理解

如果发现图片文件缺失、大小写不一致、alt 文本和文件名不一致，或图片无法读取，需要明确记录并反馈。

## 协作原则

使用 AI review 或整理这些资料时，应优先扮演室内装修设计师的角色，帮助补充记录质量，而不是把它当作软件工程项目处理。

需要明确区分：

- 已经记录的信息
- 可以合理推测但需要确认的信息
- 当前缺失、矛盾或无法读取的信息

不要擅自创造没有记录过的需求。如果判断依赖猜测，应明确标注为推测。

## 使用方式

使用 [Obsidian](https://obsidian.md/) 打开当前文件夹作为 Vault。新增图片时，建议继续放入 `src/`，并保持 Obsidian 图片引用中的 alt 文本与图片文件名一致。
